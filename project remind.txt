1. 레이아웃 만들기 (프로젝트 구성과 기능 / 제약 레이아웃 / 리니어 레이아웃 / 상대 레이아웃 
 / 프레임 레이아웃 / 기본 위젯 / 드로어블)
 - 화면을 구성하는 가장 기본적인 단위는 '뷰'
 - 뷰 중 눈에 보이는 건 '위젯' , 눈에 보이지 않는 건 '레이아웃'
 - 상태 드로어블 <selector> <item> .. state_pressed 등등 ..

2. 이벤트와 리스트뷰 (스크롤뷰 / 이벤트 처리 / 토스트와 대화상자 / 비트맵 / 인플레이션
 / 리스트뷰)
 - 나인패치(9Patch) 이미지는 특성상 깨지지 않도록 자동으로 크기 맞춰줌
 - 화면을 터치할 때 '이벤트' 발생
 - 인플레이션 이란 XML 레이아웃에 정의된 내용이 메모리에 객체화되는 과정
 - 리스트뷰 .. 어댑터 .. (뷰가 아닌 어댑터가 실제 데이터를 관리)
 - 여러 아이템 중 하나를 선택하는 선택 위젯 쓸 때 '어댑터' 사용하기

3. 화면 여러 개 만들기 (화면전환 / 인텐트 / 수명주기 / 서비스 / 브로드캐스트 수신사)
 - 시스템 이 관리하는 애플리케이션 구성요소: 액티비티, 서비스, 브로드캐스트리시버, 내용제공자
 - 시스템에서 이해할 수 있는 객체로 만든 것이 '인텐트'
 - 인텐트에 추가할 수 있는 옵션은 '플래그'
 - 시스템에서 건드리지 않고 전달되는 '부가 데이터' (인텐트 안 번들 객체로 put, get 사용)
 - 부가데이터 자료형 중 객체를 넣는 Serializable과 Paraceleable
 - 수명주기는 화면 상태에 따라 달라지며, 각 상태에 따라 자동 호출되는 메소드 상이
 - 파라미터로 전달된 번들 객체의 데이터 이용 시 화면에 원래 데이터 복구 가능
 - 화면이 없는 상태에서 실행되는 애플리케이션의 구성요소 '서비스' (인터넷 데이터 전달)
 - 여러 사람에게 동시에 데이터를 전달하는 '브로드캐스트 수신자'

7. 멀티미디어 (사진찍기 / 오디오와 동영상 / 녹음 / 리싸이클러뷰)
